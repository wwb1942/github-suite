# /github-analyzer - GitHub源码分析器

GitHub开源项目源码深度分析器。对项目进行多维度调研分析，包含架构理解、模块解析、设计模式识别、代码质量评估。

## Instructions

### 分析模式

**1. 深度调研模式 (Deep Research)**
- 完整阅读代码，多维度理解
- 至少3遍阅读：结构→逻辑→细节
- 输出完整的多维度分析报告
- 适用于：技术选型、架构学习、二次开发准备

**2. 快速概览模式 (Quick Overview)**
- 关键文件快速扫描
- 重点关注README、入口文件、核心模块
- 生成简要概要报告
- 适用于：初步评估、快速了解、筛选项目

**3. 对比分析模式 (Comparative)**
- 多个相似项目对比
- 横向对比技术选型、架构设计、功能实现
- 生成对比矩阵和推荐结论

### 工作流程

```
Step 1: 需求确认
→ 确认项目URL和分析模式
→ 如需深度调研，确认关注维度

Step 2: 项目获取
→ 克隆或浏览项目代码
→ 识别项目语言和框架

Step 3: 结构分析
→ 解析目录结构
→ 识别技术栈和依赖

Step 4: 架构分析
→ 识别架构模式
→ 分析分层设计

Step 5: 模块分析
→ 深入核心模块
→ 分析数据流和接口

Step 6: 模式识别
→ 识别设计模式
→ 评估应用质量

Step 7: 质量评估
→ 多维度评分
→ 生成质量评估卡

Step 8: 报告生成
→ 汇总分析结果
→ 提炼创新亮点和建议

Step 9: 成果存储（可选）
→ 存入调研成果库
→ 更新索引
```

### 6维度分析框架

1. **项目结构分析**
   - 目录结构、模块划分
   - 依赖关系、技术栈识别

2. **架构设计分析**
   - 整体架构模式识别（分层/微服务/事件驱动/插件等）
   - 分层设计理解、核心组件交互

3. **模块深度解析**
   - 核心模块功能
   - 数据流分析、关键算法

4. **设计模式识别**
   - 创建型：单例、工厂、建造者
   - 结构型：适配器、装饰器、外观
   - 行为型：策略、观察者、命令

5. **代码质量评估**
   - 代码风格一致性
   - 错误处理机制
   - 测试覆盖情况
   - 文档完整性

6. **创新点提炼**
   - 技术亮点总结
   - 可复用模式
   - 学习借鉴价值

### 调研成果库（可选）

分析报告可存入本地调研成果库，方便后续引用。默认建议路径：

```
~/research/github-analysis/
```

> 你可以根据自己的项目结构自定义存储路径。

**引用机制**:
当用户询问已调研项目时，优先引用调研成果库中的分析报告。

## Quick Reference

| 参数 | 说明 |
|------|------|
| 分析模式 | 深度调研 (默认) / 快速概览 / 对比分析 |
| 6 维度 | 项目结构、架构设计、模块解析、设计模式、代码质量、创新点 |
| 输出格式 | JSON 报告 + 质量评估卡 |
| 成果库 | 可自定义，默认建议 `~/research/github-analysis/` |
| 默认行为 | 未指定模式时默认"深度调研"，未指定维度时使用全部 6 维度 |
